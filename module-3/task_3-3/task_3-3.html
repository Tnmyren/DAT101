1. What is the difference between invoking and creating a function?

    2. Creating a function is when you define what the function does and what code it runs, invoking the function is when you make the function do it.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Javascript is a loosely typed language, meaning that semicolons are optional and code blocks are defined by curly braces.

  3. What happens when we try to store values from a function when it doesn't return any

    4. If the variable is defined within the function, it will be undefined outside of it.

  4. How do we separate parameters from each other when creating a function?

    5. Commas are the separators between parameters.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. During both creation and invocation, the function name is followed by brackets. Within these the placeholder variables for the parameters are defined, separated by commas. When invoked the values wil match with the variables.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. Because the function is expecting a value for that parameter, and if it is not provided, it defaults to undefined. The parameters are typically important for the function and may not work as intended without them.

  7. Explain what happens when we use recursive functions.

    8. A recursive function calls itself, eliminating the need for loops in certain cases.
